# Maeterna Agent Guide

## Workspace

- Use Node 20+ and pnpm 10.33.4. Install dependencies with `pnpm install` (or `vp install`).
- This workspace contains `apps/web` (React SPA) and `apps/api` (Hono Cloudflare Worker with D1). `packages/*` is configured as a workspace glob but no shared package currently exists.
- Start both apps from the root with `pnpm dev`. Do not run `vp dev` at the root; it is tooling configuration, not a Vite app. Target the web app with `pnpm --filter ./apps/web dev` and the API with `pnpm --filter maeterna-api dev`.
- Run `vp check` from the root for formatting, linting, and type checking. `vp check --fix` applies formatting fixes. Staged files are checked with `vp check --fix`.
- The formatter requires LF, two spaces, double quotes, no semicolons, 80-column lines, and trailing ES5 commas.
- If no spec doc has been supplied for a feature, output your plan to a spec document in `docs/specs` and give the file a name that increments from the numerical prefix of the last file.
- When a ticket is complete, update this file if applicable. DO not just append to it. Update it to ensure that is current.
- When a ticket is complete, also update `README.md`, `apps/web/README.md` and `apps/api/README.md` if applicable.

## Web

- `apps/web` is a client-only TanStack Router SPA. Routes in `src/routes/` generate `src/routeTree.gen.ts`; never edit the generated tree.
- The web app needs `apps/web/.env.local` with `VITE_API_URL`, `VITE_BETTER_AUTH_URL`, and `VITE_APP_URL`; its API and auth server default to `http://localhost:8787` during local development.
- Use `apiClient` from `src/lib/api-client.ts` for server data and `useSession()` from `src/lib/session.ts` for cached session reads. Do not add application behavior to `src/mock/`; it is legacy mock data retained mainly for type imports.
- `src/lib/api.types.ts` is generated by `openapi-typescript`. Regenerate it from a running API's `/openapi.json`, never from `apps/api/maeterna-openapi.yaml`, which is a drifted reference document.
- `/patient/*` is mobile-first with a constrained layout and bottom navigation; `/doctor/*` and `/admin/*` use desktop layouts. Their top-level `route.tsx` files enforce role-specific authentication.

## API

- Read `apps/api/CLAUDE.md` before changing the API. It documents local credentials, authentication behavior, and database workflow.
- API routes use `@hono/zod-openapi` schemas under `apps/api/src/routes/` and are registered in `src/index.ts`. Those runtime schemas generate the authoritative `/openapi.json`; update `maeterna-openapi.yaml` only as documentation when a contract changes.
- The D1 schema is `apps/api/src/db/schema.ts`; generated migrations are `apps/api/drizzle/migrations/`. Run `pnpm --filter maeterna-api db:generate` after schema changes.
- `pnpm --filter maeterna-api db:migrate` mutates the remote D1 database. Do not run it for local development or verification. `wrangler dev` provisions local D1; seed it with `pnpm --filter maeterna-api db:seed:local`.
- After changing `apps/api/wrangler.jsonc`, run `pnpm --filter maeterna-api cf-typegen` to refresh `CloudflareBindings`.
- Before writing helper functions, check `src/lib` to see if one already exists
